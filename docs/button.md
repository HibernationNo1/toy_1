========================================================
저연령·초보자 친화 UI 컴포넌트 강제 명세(Contract)
- 공통 핵심: (1) 텍스트 최소 (2) 다음 행동 1개만 강조
            (3) 입력→피드백(≤0.1s) (4) 실패/지연에도 무반응 금지
            (5) 버튼 동시 노출 ≤3
========================================================


--------------------------------------------------------
[0. 모든 컴포넌트 공통 필수]
--------------------------------------------------------
[필수 옵션/특성]
- Visible 초기값 명시 (true/false)
- ZIndex 규칙 고정: Base=1~10 / Panel=20~40 / Overlay=80~90 / Popup=100+
- AnchorPoint 고정(권장 0.5,0.5) + Position 규칙 일관
- Size 정책: 모바일 기준 터치 영역 최소 44x44px(버튼류)
- AutoButtonColor=false(버튼류) : 상태 표현은 직접 제어(일관성 확보)
- UI 상태(State) 필수: Idle / Pressed / Disabled (+ 선택: Hover/Selected)
- “행동 가능 상태만 보이기”: 사용할 수 없으면 숨기거나 Disabled(설명 금지)
- 텍스트 사용 시: 1~2 단어, 설명 문장 금지

[필수 동작]
- 입력 즉시 1차 피드백(≤0.1s): (시각) + (청각) 최소 1개는 반드시
- 성공/실패/불가에도 무반응 금지:
  - 성공: 밝은 플래시/상승음
  - 실패/불가: 짧은 흔들림/둔한 짧은 음(불쾌 금지)
- 지연 분리: 로컬 1차 반응은 즉시, 서버 결과(2차 반응)는 늦어도 됨
- 위치/모양/반응 규칙성: 같은 행동은 항상 같은 위치/같은 피드백

========================================================
1) 버튼 만들기 (ImageButton 사용)
========================================================

--------------------------------------------
1-1. 닫기 버튼 (BtnClose)
--------------------------------------------
[필수 옵션/특성]
- 위치 고정(우상단/좌상단 중 1개 표준) : 화면마다 이동 금지
- 아이콘만 사용(텍스트 제거) + 오입력 방지 위해 히트박스 크게
- 배경/테두리는 “덜 강조”(주 행동 버튼과 경쟁 금지)
- Disabled 상태에서도 시각 유지(숨기기보다 실수 방지용)

[필수 동작]
- OnClick: 즉시 Close(팝업/패널) 요청
- 1차 피드백: 눌림 스케일 다운 + 클릭 사운드
- 실수 보호: 1) 취소 가능 구조 유지 2) “되돌릴 수 없다” 느낌 금지
- 연속 입력: 0.2s 쿨다운(더블 클릭 방지)

--------------------------------------------
1-2. 확인 버튼 (BtnConfirm)
--------------------------------------------
[필수 옵션/특성]
- “다음 행동 1개 강조”의 대표 버튼(Primary)
- 화면 내 동시 노출 시 가장 크고 눈에 띄어야 함
- 아이콘+짧은 단어(선택) 가능하지만 설명 금지
- 로딩/대기 상태 표현 슬롯(작은 스피너/점멸) 반드시 확보

[필수 동작]
- OnClick: 연타 방지(즉시 Disabled 또는 Pending)
- 1차 피드백: 강한 시각(밝기/테두리) + 성공음(짧게)
- 서버 지연 시: “진행 중”을 느낌으로(점멸/작은 움직임) 처리
- 실패 시: 경고 팝업 금지, 대신 “약한 실패 피드백 + 즉시 재시도” 유지

--------------------------------------------
1-3. 취소 버튼 (BtnCancel)
--------------------------------------------
[필수 옵션/특성]
- Confirm보다 덜 강조(Secondary) : 선택 부담 최소화
- Confirm과 높이/패딩 동일(쌍으로 쓰일 때 안정감)

[필수 동작]
- OnClick: 즉시 닫기/이전으로 복귀
- 실패/불가 개념 없음(취소는 항상 가능해야 함)
- 1차 피드백: Confirm보다 약하게(짧은 소리/약한 플래시)

--------------------------------------------
1-4. 뒤로가기 버튼 (BtnBack)
--------------------------------------------
[필수 옵션/특성]
- “통제감” 제공용 핵심: 항상 같은 위치(좌상단/헤더)
- 아이콘 중심(텍스트 최소화)
- 기능 버튼들과 경쟁 금지(Primary와 시각적으로 분리)

[필수 동작]
- OnClick: 화면 스택 Pop(즉시)
- 스택 없음일 때: 무반응 금지 → “약한 실패 피드백(톡/흔들림)”만 주고 유지
- 연속 입력 방지: 0.2~0.3s 쿨다운

--------------------------------------------
1-5. 탭 버튼 (BtnTab)
--------------------------------------------
[필수 옵션/특성]
- 선택/비선택 상태가 한눈에(색/밝기/밑줄/크기 중 1개는 강제)
- 탭은 “선택지”가 아니라 “현재 모드” 표시 → 과한 장식 금지
- 탭이 많아지면 규칙 위반 위험(버튼 ≤3 원칙) → 기본은 2~3 탭까지만

[필수 동작]
- OnClick: TabChanged 이벤트 발행(컨텐츠 직접 조작 금지)
- 이미 선택된 탭 클릭: 기본은 무시(무반응 금지 → 아주 약한 클릭 피드백만)
- 탭 전환 시: 컨텐츠 변화는 “유저 입력 결과”로만 발생(자동 전환 금지)

========================================================
2) 카드 만들기 (Frame 사용)
========================================================

--------------------------------------------
2-1. 아이템 카드 (CardItem)
--------------------------------------------
[필수 옵션/특성]
- 텍스트 최소: 이름은 짧게, 설명문 금지
- “읽지 않아도 구분”되게: 희귀도=색/테두리/배지 1개로 고정
- 오입력 대비: 카드 전체가 큰 히트 영역(클릭 가능 시)
- 상태 표현:
  - Selected(선택됨) : 시각적으로 확실
  - Locked(잠김) : “불안” 대신 “아직 안 됨” 느낌(부드럽게)
- 결과는 중앙: 카드 클릭 결과(선택됨/장착됨)는 화면 중앙 근처 피드백으로

[필수 동작]
- SetItem(data): icon/rarity/qty 등 바인딩
- OnClick: “상세 보기”나 “선택” 중 1개만 연결(동시에 두 행동 금지)
- 실패(없음/잠김)에서도 무반응 금지:
  - 잠김이면 카드가 살짝 흔들리며 둔한 짧은 음

--------------------------------------------
2-2. 아이템 등장 카드 (CardItemReveal)
--------------------------------------------
[필수 옵션/특성]
- 목적: “성공 체감/획득 확신” (설명 대신 연출)
- 연출은 짧아야 함: 0.5~1.5초 내 종료, 스킵 가능
- 희귀도 연출 강도는 “과장 허용” (초반 확신이 우선)
- 표시 요소는 최소:
  - 큰 아이콘(또는 ViewportFrame)
  - 희귀도 색/광
  - (선택) NEW 배지

[필수 동작]
- PlayReveal(data): 등장 애니메이션(Scale/Glow) + 획득음
- Skip(): 누르면 즉시 종료(저사양/지연 대응)
- 다음 행동 1개만 강조:
  - 예: “다시 뽑기” 또는 “닫기” 중 1개만 강하게

========================================================
3) 판넬 만들기 (Frame 사용)
========================================================

--------------------------------------------
3-1. 그리드 판넬 (PanelGrid)
--------------------------------------------
[필수 옵션/특성]
- UIGridLayout 강제(셀 크기/간격 표준화) → “정돈”이 곧 이해
- 항목이 많으면 ScrollingFrame 버전(PanelGridScroll) 필수
- EmptyState(비어있음) 표시: 텍스트 대신 아이콘/간단 단어(“없음”)만
- 아이템 클릭 결과는 중앙 피드백으로(선택/장착/획득)

[필수 동작]
- SetItems(list): 재사용(전체 Destroy/Recreate 남발 금지)
- 입력 지연 시: 스크롤/클릭은 로컬 피드백 유지, 데이터는 늦게 반영 가능

--------------------------------------------
3-2. 가로 리스트 판넬 (PanelListH)
--------------------------------------------
[필수 옵션/특성]
- UIListLayout Horizontal + Padding 고정
- “선택 부담” 줄이기 위해 화면에 동시에 보이는 항목 수 제한(너무 많으면 고민 증가)
- 스크롤이 있으면 “당겨서 더 있음”을 느낌으로(살짝 잘린 카드) 표현

[필수 동작]
- SetChildren(list)
- ScrollToIndex(i)(선택): 탭/선택 연동 시 즉각 이동(연출 짧게)

--------------------------------------------
3-3. 세로 리스트 판넬 (PanelListV)
--------------------------------------------
[필수 옵션/특성]
- UIListLayout Vertical
- 항목 높이 규칙성 유지(리스트는 읽게 만들기 쉬움 → 읽기 비용 최소화)
- 섹션 구분은 텍스트 대신 간단 라인/색 블록

[필수 동작]
- SetChildren(list)
- LoadMore(선택): “더 보기”는 버튼 추가 대신 자동 로딩/하단 자연 확장 권장

--------------------------------------------
3-4. 탭 컨텐츠 판넬 (PanelTabContent)
--------------------------------------------
[필수 옵션/특성]
- 탭 수는 기본 2~3개(버튼 ≤3 원칙 준수)
- 탭 바는 상단 고정, 컨텐츠만 교체(위치 변화 금지)
- 탭 전환 시 “자동 팝업/설명” 금지

[필수 동작]
- SetTabs(defs), SetActive(key)
- 전환 피드백: 아주 짧은 소리/밝기 변화(강조는 ‘현재 탭’만)

--------------------------------------------
3-5. OX 선택 판넬 (PanelChoiceOX)
--------------------------------------------
[필수 옵션/특성]
- 저연령에게 “되돌릴 수 없음” 공포 유발 가능 → 표현을 부드럽게
- 질문 텍스트는 1문장 이하, 최대한 단순
- 버튼은 2개지만 “정답 1개 강조” 규칙을 적용:
  - 기본 추천값(권장) 하나를 더 크게/밝게(예: 안전한 선택)
- 실수 방지: 버튼 간 간격 충분히

[필수 동작]
- Open(question)
- OnChoose(true/false)
- 실패/불가 없음: 두 선택 모두 항상 눌려야 함
- 선택 후: 긴 확인 팝업 금지 → 즉시 결과 피드백 + 다음 행동 1개 강조

--------------------------------------------
3-6. 팝업 판넬 (보기용: 선택/클릭 없음) (PanelPopupView)
--------------------------------------------
[필수 옵션/특성]
- “보기 전용”이면 진짜로 입력 요소(버튼/슬라이더) 제거
- 텍스트 설명 남발 금지:
  - 핵심은 “보인다/느껴진다”
- 팝업은 “유저 입력 결과로만” 등장(자동 팝업 금지)

[필수 동작]
- Open(data), Close()
- 닫기 동작은 별도 Close 버튼(상단 고정) 또는 Back 키로 제공
- 지연/오류 상황에서도 무반응 금지:
  - 내용 로딩 중이면 “점멸/아이콘”으로 진행 느낌 제공

  ------------------------------------------
  3-6-a. 오버레이 팝업창 (OverlayView)
  ------------------------------------------
  [필수 옵션/특성]
  - 배경 Dim(불안 유발 없는 수준의 어두움) + 입력 차단(Blocker)
  - 닫기 유도는 “1개만”: 닫기 버튼 또는 Back키 (둘 다 강하게 강조 금지)
  - 등장/퇴장은 짧게(0.15~0.25초)

  [필수 동작]
  - Show(): Dim-in + 짧은 등장음(선택)
  - Hide(): Dim-out
  - 배경 탭 정책:
    - 보기용이면 기본은 “무시”(실수로 닫혀서 불안/혼란 방지)
    - 닫기를 허용한다면: ‘확실한 피드백’과 함께 닫힘

  ------------------------------------------
  3-6-b. 상세보기 팝업창 (DetailView)
  ------------------------------------------
  [필수 옵션/특성]
  - 목적: “설명”이 아니라 “확신/비교” (정보를 읽게 만들지 말 것)
  - 큰 프리뷰(아이콘/3D) + 희귀도 색 + 핵심 수치 1~2개만
  - 여러 버튼/행동 금지: 상세보기 안에서 다음 행동은 1개만(예: 장착 OR 판매)

  [필수 동작]
  - SetTarget(data): 패널 재생성 대신 내용만 교체(일관성/성능)
  - Close(): 즉시 복귀(정지 화면/확인 단계 금지)
  - 선택 결과 피드백은 중앙에서 발생(장착됨/등록됨 같은 짧은 표시)
